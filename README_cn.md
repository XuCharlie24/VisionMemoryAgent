# Vision Memory Agent 视觉记忆智能体

Vision Memory Agent 是一个基于 RDK X3 开发板的视觉记忆智能体系统。

本项目通过摄像头实时采集环境画面，在 RDK X3 板端完成图像获取、目标识别、视觉记忆记录与状态管理，并在 PC 端通过 Web 前端进行可视化展示和交互。

系统希望实现一个轻量级的视觉记忆智能体：它能够观察环境、识别物体、记录出现过的目标，并通过前端界面展示当前记忆状态，同时支持一定的人机交互能力。

## 核心功能

- 实时摄像头画面采集
- 视频流实时展示
- 常见目标识别
- 视觉记忆记录
- 目标出现次数统计
- 目标位置与状态更新
- 手势交互
- Web 可视化界面
- 前后端分离架构
- 基于 RDK X3 的边缘端部署

## 系统架构

本项目采用前后端分离架构。

后端运行在 RDK X3 开发板上，摄像头连接到 RDK X3。后端负责摄像头读取、图像处理、目标识别、视觉记忆管理和接口服务。

前端运行在 PC 浏览器中，通过 HTTP 接口访问后端视频流、识别结果和记忆状态。用户可以在前端界面中查看实时画面、目标识别结果和视觉记忆列表。

## 功能模块说明

## 1. 视频采集模块

系统通过摄像头实时采集环境画面，并由后端提供视频流接口，前端可以直接展示实时画面。

## 2. 目标识别模块

后端对摄像头画面进行分析，识别场景中的常见目标，例如水杯、手机、书本、瓶子、键盘、鼠标和人体等。

## 3. 视觉记忆模块

系统会对识别到的目标进行记忆记录，包括目标类别、置信度、位置、出现次数和最近状态等信息。

为了避免同一目标被频繁重复记录，系统会根据类别和位置进行简单去重，并维护一个轻量级视觉记忆列表。

## 4. 手势交互模块

系统支持一定的手势交互能力，用户可以通过手势与前端界面进行交互，使系统更接近自然人机交互场景。

## 5. 可视化展示模块

前端提供可视化控制台，用于展示实时视频画面、识别结果、记忆列表、状态卡片和三维风格交互效果。

## 后端启动方式

进入后端目录：

cd backend

安装依赖：

pip3 install -r requirements.txt

启动后端服务：

python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000

启动成功后，后端服务地址为：

http://<RDK_X3_IP>:8000

其中 <RDK_X3_IP> 需要替换为实际 RDK X3 开发板的 IP 地址。

## 前端启动方式

进入前端目录：

cd frontend

安装依赖：

npm install

启动前端开发服务：

npm run dev

启动完成后，根据终端输出的地址，在浏览器中打开前端页面。

## 常用接口说明

后端常用接口包括：

GET /api/video/stream
GET /api/memory/status
POST /api/memory/reset
GET /api/status/current

实际接口路径以本仓库后端代码实现为准。

## 部署流程

1. 将 USB 摄像头连接到 RDK X3 开发板。
2. 在 RDK X3 上进入 backend 目录。
3. 安装后端依赖并启动 FastAPI 服务。
4. 在 PC 端进入 frontend 目录。
5. 安装前端依赖并启动前端页面。
6. 在浏览器中访问前端地址。
7. 查看实时视频、目标识别结果和视觉记忆状态。
